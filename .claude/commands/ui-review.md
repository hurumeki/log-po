以下の手順でUIレビューを実施してください。

## 手順

1. ビルドとスクリーンショット撮影を実行:
   ```bash
   npm run build && npx playwright test e2e/ui-review.spec.js
   ```

2. `e2e/screenshots/` の全画像を Read ツールで確認する

3. 以下の観点でUIの問題を洗い出し、P0〜P3の優先度を付ける:
   - **P0 (ブロッカー)**: ボタンが隠れる・操作不能（z-index、overflow）
   - **P1 (高)**: タッチターゲット不足（44x44px未満）、コントラスト不足
   - **P2 (中)**: 完了状態のフィードバック不足、レイアウト崩れ
   - **P3 (低)**: スペーシング、フォントの一貫性

4. 指摘事項を以下の形式でまとめる:
   - 優先度 (P0-P3)
   - 該当画面（スクリーンショット名）
   - 問題の詳細
   - 該当ファイル:行番号
   - 修正案

5. P0〜P2の問題を修正する

6. 修正後、再度スクリーンショットを撮影して確認する:
   ```bash
   npm run build && npx playwright test e2e/ui-review.spec.js
   ```
