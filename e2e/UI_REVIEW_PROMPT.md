# UI Review Prompt Template

以下のプロンプトをClaude Codeに貼り付けて、UIレビューを実行してください。

---

## プロンプト

```
UIレビューを実施してください。以下の手順で進めてください:

1. `npm run build` でビルド
2. `npx playwright test e2e/ui-review.spec.js` でスクリーンショットを撮影
3. `e2e/screenshots/` の全画像を確認し、以下の観点でレビュー

### レビュー観点
- **P0 (ブロッカー)**: 操作不能な箇所（ボタンが隠れる、クリックできない等）
- **P1 (高)**: タッチターゲットの不足（44x44px未満）、コントラスト不足で見えない要素
- **P2 (中)**: 完了状態の視覚的フィードバック不足、レイアウトの崩れ
- **P3 (低)**: 細かいスペーシング、フォントの一貫性

### レビュー対象画面
- ミッション画面（空状態 / データあり / 完了状態）
- ミッション追加モーダル
- カレンダー画面
- ご褒美画面（空状態 / データあり）
- ご褒美追加モーダル
- 設定画面

### 出力フォーマット
各指摘について以下を記載:
- 優先度 (P0-P3)
- 該当画面 / スクリーンショット名
- 問題の詳細
- 該当ファイル:行番号
- 修正案

レビュー後、P0-P2の指摘を修正してください。
```

---

## スクリーンショットだけ撮り直す場合

```bash
npm run build && npx playwright test e2e/ui-review.spec.js
```

## テスト対象デバイスを変更する場合

`playwright.config.js` の `projects` を編集:

```js
projects: [
  { name: 'iPhone 14', use: { ...devices['iPhone 14'] } },
  { name: 'Pixel 7', use: { ...devices['Pixel 7'] } },
],
```
